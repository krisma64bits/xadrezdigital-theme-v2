1. Qualquer valor arbitrário como w-[500px], text-[24px], p-[15px] é PROIBIDO. Use apenas as classes padrão do Tailwind.
2. Sempre Tailwind first, nunca CSS puro!
3. Mantenha o design consistente com o design-token.json
4. Mantenha o código limpo e organizado, siga as boas práticas de programação. Siga DRY (Don't Repeat Yourself).
5. Mantenha o tema 100% compatível com Woocommerce e Plugins de terceiros.
6. Mantenha o tema 100% compatível com o WordPress.
7. O tema não será compatível com Elementor ou Gutenberg.
8. Ele não será distribuido, é exclusivo para o Xadrez Digital.
9. O tema não deve ter codigo sujo, ele deve ser limpo e organizado.
10. O tema deve ser responsivo e mobile first.
11. Focado em altíssima perfomance, tentando o melhor desempenho possível.
12. O tema deve ser otimizado para SEO.
13. O tema deve ser otimizado para o Google Lighthouse.
14. Não remova estilos do Woocommerce ou wordpress sem que o usuário peça.
15. Ícones devem ser armazenados como arquivos .php individuais em inc/icons/ (ex: cart.php, whatsapp.php). Use get_template_part('inc/icons/nome-do-icone') para incluir. Suporte a classes customizadas via $args['class'].
16. Customização WooCommerce com compatibilidade: Para estilizar componentes WooCommerce (tabs, forms, etc) sem quebrar compatibilidade com plugins:
    a) Use filtros/hooks para INJETAR classes customizadas no wrapper (ex: output buffering + str_replace para adicionar classe)
    b) Use CSS Tailwind (em src/components/) para estilizar via classe injetada
    c) NUNCA faça template override apenas para mudar estilos
    d) NUNCA altere a estrutura HTML/markup original do WooCommerce
    e) Mantenha atributos ARIA e funcionalidade JavaScript intactos
